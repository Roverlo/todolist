## 目标
- 附件移动到第二行，与状态/优先级/截止日期/责任人/创建时间同排；各字段在第二行均匀分布
- 第二行自适应地承载所有后续新增列（extras），无需改代码即可显示
- 第二行不同字段采用柔和配色区分模块，增强可读性

## 布局与字段
- 第1行（保持）：项目、标题、详情、最近进展、下一步、操作
- 第2行（统一模块）：状态、优先级、截止日期、现场责任人、产线责任人、创建时间、附件、所有自定义列（extras）
- 显示规则：
  - 基础字段始终显示（第二行）
  - 自定义列（extras）只在有值或处于编辑态时显示

## 技术实现
### 1. 字段集合
- 在 `TaskTable.tsx` 统一定义：
  - `TOP_COLUMNS = ['project','title','notes','latestProgress','nextStep']`
  - `BOTTOM_BASE_FIELDS = ['status','priority','dueDate','onsiteOwner','lineOwner','createdAt','attachments']`
- 第二行渲染：`BOTTOM_BASE_FIELDS.concat(dynamicExtras)`，其中 `dynamicExtras = columns.columns.filter(c => !TOP_COLUMNS.includes(c) && !BOTTOM_BASE_FIELDS.includes(c))`

### 2. 自适应栅格
- `.row-bottom { display: grid; grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)); gap: 8px; }`
- 媒体查询：1120px 以下改为 4 列，768px 以下改为 2 列，保证匀称分布

### 3. 彩色模块
- 每个字段块渲染 `class="field module module--{key}"`
- 样式颜色方案（柔和浅色背景）：
  - `status`：淡黄 `#fff7cc`
  - `priority`：淡红 `#ffe0e0`
  - `dueDate`：淡蓝 `#e6f0ff`
  - `onsiteOwner/lineOwner`：淡绿 `#f0fdf4`
  - `createdAt`：淡灰蓝 `#f1f5f9`
  - `attachments`：淡紫 `#f3e8ff`
  - `extras`：淡青 `#e6fffa`
- `.module { border-radius: 8px; padding: 8px; } .label { font-size: 12px; color: #64748b }`

### 4. 编辑与展示
- 编辑态沿用现有 `renderEditor`；展示态沿用 `renderDisplay`
- 自定义列在编辑态展示输入框，保存写入 `task.extras[col]`

### 5. 其它对齐
- 第二行 `colSpan = TOP_COLUMNS.length + 2` 保持与表头一致
- 仅项目/标题 sticky；其余不 sticky，减少横向滚动

## 验证
- 第二行均匀分布并自适应；附件与所有扩展列都在第二行
- 彩色模块清晰区分；无空白块（extras仅在有值或编辑时显示）
- 绿色版运行检查宽屏/窄屏表现
