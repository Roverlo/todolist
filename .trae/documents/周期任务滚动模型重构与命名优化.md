## 目标
- 把周期任务从“可配置很多参数”简化为“极少的选择 + 明确默认值”。
- 去掉锚点日期、灵活/严格、复杂的间隔逻辑；保留最常用的每周/每月场景。
- 按你的直觉交互：填写信息 → 点击“生成周期任务”；完成后可自动续期。

## 简化后的模型
- 周期选项：
  - 周期类型：每周 | 每月（二选一）
  - 执行日：
    - 每周：周一–周日（默认：今天所在周的最近一个工作日）
    - 每月：1–31（默认：当月今日，溢出自动对齐月底）
  - 截止日期策略：周末 | 月底 | 当天（默认：周末/月底）
- 自动续期：复选框“完成后自动续期”（默认勾选）
  - 勾选时：任务标为“已完成”会自动生成下一期任务（新任务），截止日期按下一周/下月计算
  - 不勾选：仅生成当期一次，不自动续期
- 命名优化：弹窗按钮统一为“生成周期任务”

## 行为细则
- 生成：
  - 一次点击生成一个或多条（如果一周选择了多个执行日，则按选择的周几一次性生成多条；每月只生成一条）
  - 生成后自动切换筛选到该项目并显示所有状态
- 自动续期（勾选时）：
  - 当某条周期任务标记为“已完成” → 立即新建下一期任务，复制标题/负责人/默认字段，截止日期改为下一周期截止时间
  - 不修改历史任务：上一期保持“已完成”
  - 不需要锚点和间隔，永远按“下一周/下个月”推

## 技术改动
- UI
  - RecurringTaskModal：
    - 移除锚点日期、灵活模式、间隔等字段
    - 保留周期类型、执行日、截止策略、自动续期勾选
    - 按钮文案改为“生成周期任务”
- 生成逻辑
  - 直接 `addTask`：按当前选择的周几/月日计算当期日期与截止日期并新建任务
  - 生成后 `setFilters({ projectId, status: 'all', statuses: [] })`
- 自动续期
  - 在 `updateTask`（或监听状态变化）检测周期任务且“完成后自动续期”为真：
    - 计算下一周/下月的日期与截止策略
    - 新建下一期任务（不是复用同一条），并在完成的那条上追加一条进展“已续期到下一期”
- 数据结构
  - 在周期任务的 `extras` 中标记：`extras.recurring = { type: 'weekly'|'monthly', day: number, dueStrategy: 'endOfWeek'|'endOfMonth'|'sameDay', autoRenew: boolean }`
- 启动补偿（可选，简化版）
  - 开启应用时，如果存在“已完成且自动续期”为真但尚未生成下一期的任务，补一道续期（避免关闭期间遗漏）

## 成功标准
- 用户只需要 3 个选择 + 1 个勾选，就能生成周期任务
- 标记完成后是否自动续期一目了然；下一期任务总能按周/按月正确出现
- 无需理解“锚点”“间隔”“灵活”等概念
